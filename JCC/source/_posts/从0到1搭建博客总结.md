## 一、搭建前的准备

搭建前需要下载这些软件：

Git：官网下载：[https://git-scm.com/](https://git-scm.com/)
Node.js 官网下载：[http://nodejs.cn/](http://nodejs.cn/)

## 二、安装 hexo

1.Hexo 是一款基于 Node.js 的快速、简洁且高效的静态博客框架。在安装好 Node.js 后进入 cmd 命令行 输入指令安装 hexo

```csharp
npm install -g hexo-cli
```

2.新建一个文件夹放 hexo 项目，然后右键打开 git bash

![](./EN2jbXD8SorQ2IxaRLKcNhQMnns.png)

输入指令创建 hexo 项目，如果在创建过程中报错，可能是网络问题可以挂个梯子清空文件夹再创建一次。

```csharp
hexo init
```

3.输入指令启动 hexo 项目

```csharp
hexo s
```

![](./LIJ4bgFdVoesEBx6SPSckYItnJe.png)

新建的项目中有：

node_modules: 依赖包
public：存放生成的页面
scaffolds：生成文章的一些模板
source：用来存放你的文章
themes：主题
** _config.yml: 博客的配置文件**

4.在本地尝试将网站打开

```csharp
http://localhost:4000/
```

![](./COQXbjyTko7FEpxRnB3cX6QjnHV.png)

出现这个页面说明就成功了。

## 三、将 Hexo 项目部署到 Github 上

将 Hexo 项目部署到 GitHub 上后就可以在多台设备上访问我们网站了。部署流程如下：

### 1.在 GitHub 上新建仓库

新建的仓库名必须是你的用户名.github.io，否则仓库名就是无效的无法部署成功。

![](./MAMebQOOfocL3BxSkbcc6nUDn0b.png)

### 2.生成 ssh 添加到 Github

ssh 提供了安全验证和免密操作，hexo 项目与 github 远端仓库交互时不需要每次输入用户名和密码。

第一步：配置 git 中的用户名和邮箱，打开 cmd, 输入以下命令

```csharp
git config --global user.name "yourname"
git config --global user.email "youremail"
```

这里的 yourname 输入你的 GitHub 用户名，youremail 输入你 GitHub 的邮箱。这样 GitHub 才能知道你是不是对应它的账户。用户名为仓库的名称，邮箱为注册 github 的邮箱

第二步： 在 hexo 项目的目录下 右键打开 git bash 输入指令 生成公钥

```csharp
ssh-keygen -t rsa -C 自己的邮箱地址
```

输入指令后 连敲击三次回车 出现下图就成功了

![](./UaAdbHpsmoJbwOxdBZYcZjRinrx.png)

第三步，创建的公钥在.ssh 目录中，打开该目录 id_rsa.pub 表示公用复制这里面的内容，然后进入 github.

点击 setings

![](./RxHMbKKxNorNJCx4KP3c6Od3nCf.png)

![](./AvMkbUV27o0xl0xAqRIcMDxVnSe.png)

发现我们需要一个密钥，把我们刚刚复制的密钥粘进去，title 随便起

![](./BhxTbt17jo45Y5xnncpc2sKRnef.png)

### 3.部署到 github 中

这一步，我们就可以将 hexo 和 GitHub 关联起来，也就是将 hexo 生成的文章部署到 GitHub 上。

第一步 修改配置文件

![](./JPX6bwED8oO4hxxRrpIcy8ECnzd.png)

```csharp
修改内存
deploy:
  type: git
  repo: git@github.com:goubin18/goubin18.github.io.git //（用自己的仓库路径代替）
  branch: main
```

![](./HZpAbNBavoiLvPxyrnrcgYyZnEe.png)

第二步安装 deploy-git

```csharp
npm install hexo-deployer-git --save
```

![](./TMsyb5dcBoDpjzxQzjpctPMCnch.png)

第三步 生成静态界面并推送到远端

```csharp
hexo c   #清除缓存文件 db.json 和已生成的静态文件 public
hexo g       #生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)
hexo d       #自动生成网站静态文件，并部署到设定的仓库(hexo deploy 的缩写)
```

注意 deploy 时会让输个 yes

**最后就可以使用 xxx.github.io 来访问你的博客啦**

![](./IwWBbpmvxoGSwcxmSLtcEWs0nJd.png)
